# CyberCafe Manager — Project Brief

## Продукт

Desktop-приложение для управления компьютерным клубом на Windows.
Написано на **Electron + React + SQLite (better-sqlite3)**.

Основная идея: существующие системы (Netadmin и др.) не умеют считать кассу киоска — воду, чипсы и прочие товары. Это приложение закрывает этот пробел и добавляет полноценную кассу, склад и отчётность.

## База данных (SQLite)

Файл БД: `%APPDATA%\cybercafe-manager\cybercafe.db`

### Таблицы

- **settings** — key/value настройки (название клуба, валюта)
- **rooms** — комнаты (VIP, Комфорт, Стандарт и др.), цвет, тип
- **computers** — компьютеры, привязаны к комнате, тариф сом/час
- **products** — товары склада (название, цена, себестоимость, остаток, фото base64, скидка %)
- **product_categories** — категории товаров (Напитки, Еда, Снеки, Прочее)
- **orders** — счета (статус, комната, суммы, скидка, даты открытия/закрытия)
- **order_computers** — компьютеры в счёте (название, часы, сумма)
- **order_items** — товары в счёте (продукт, количество, цена, итог)

## Реализованный функционал (v1.0)

### Касса (`Cashier.js`)
- Создание нового счёта
- Выбор комнаты
- Добавление компьютеров (из справочника или вручную) + ручной ввод суммы
- Быстрые кнопки товаров с фильтром по категориям и поиском
- Управление количеством товаров в счёте
- Скидка % на весь счёт
- Примечание к счёту
- Итоговая сумма с разбивкой (компьютеры / товары / скидка)
- Сохранение как "открытый" или сразу закрыть (оплата)
- Редактирование открытого счёта
- Закрытие счёта (оплата) и отмена
- Объединение нескольких счетов в один
- Три вкладки: Открытые / Закрытые / Отменённые

### Статусы счетов
- `open` — открыт
- `closed` — закрыт/оплачен
- `cancelled` — отменён

### Склад (`Warehouse.js`)
- Добавление/редактирование/удаление товаров
- Поле фото (загрузка с диска, хранится как base64)
- Категория, единица измерения, цена продажи, себестоимость
- Скидка % на товар
- Остаток на складе (автоматически уменьшается при продаже, восстанавливается при отмене)
- Статус активен/скрыт
- Сводная статистика: стоимость склада, себестоимость, потенциальная прибыль
- Фильтр по категориям и поиск

### Отчётность (`Reports.js`)
- Периоды: Сегодня / Вчера / Неделя / Месяц / Прошлый месяц / Свой период
- Метрики: выручка, компьютеры, товары, количество счетов
- Bar chart по дням (компьютеры + товары)
- Pie chart структуры выручки
- Топ-10 товаров с прогресс-барами
- Детализация по дням (таблица)

### Дашборд (`Dashboard.js`)
- Фильтр: Сегодня / Неделя / Месяц
- 4 карточки метрик
- Area chart выручки по дням (итого + компьютеры + товары)
- Топ-6 товаров
- Список открытых счетов

### Настройки (`SettingsPage.js`)
- Название клуба и валюта
- Управление комнатами (CRUD, цвет, тип)
- Управление компьютерами (CRUD, привязка к комнате, тариф)

## API (preload.js → main.js)

Все вызовы через `window.api.*`:

```
window.api.getSettings() / setSetting(key, value)
window.api.getRooms() / createRoom() / updateRoom() / deleteRoom()
window.api.getComputers() / createComputer() / updateComputer() / deleteComputer()
window.api.getProducts() / getAllProducts() / createProduct() / updateProduct() / deleteProduct()
window.api.getProductCategories()
window.api.getOrders(filter) / getOrderById(id) / createOrder() / updateOrder()
window.api.closeOrder(id) / cancelOrder(id) / mergeOrders(ids)
window.api.getReportsSummary(dateFrom, dateTo)
```

## Интеграция с Netadmin (будущее)

Архитектура готова: в `main.js` есть слой Adapter API.
План: добавить `src/main/adapters/netadmin.js` с методами получения данных
по времени и ценам за компьютеры. Основная логика приложения не изменится.

При интеграции появятся статусы "На паузе" и "Ожидает оплаты" в счетах —
время будет считаться автоматически из системы клуба.

## Запуск и сборка

```bash
# Разработка
npm install
npm start

# Сборка .exe installer для Windows
npm run build:react
npx electron-builder --win
# Результат: dist/CyberCafe Manager Setup.exe
```

**Важно:** требуется better-sqlite3 v12.8.0+ для совместимости с Node.js v24.
